25  /**
26  * @brief This class generates and stores adaptive finite element meshes for the real-space dft problem.
27  *
28  * The class uses an adpative mesh generation strategy to generate finite element mesh for given domain
29  * based on five input parameters: BASE MESH SIZE, ATOM BALL RADIUS, MESH SIZE ATOM BALL, MESH SIZE NEAR ATOM
30  * and MAX REFINEMENT STEPS (Refer to utils/dftParameters.cc for their corresponding internal variable names).
31  * Additionaly, this class also applies periodicity to mesh. The class stores two types of meshes: moved
32  * and unmoved. They are essentially the same meshes, except that we move the nodes of the moved mesh
33  * (in the meshMovement class) such that the atoms lie on the nodes. However, once the mesh is moved, dealii
34  * has issues using that mesh for further refinement, which is why we also carry an unmoved triangulation.
35  * There are other places where we require an unmoved triangulation, for example in projection of solution
36  * fields from the previous ground state in stucture optimization.
37  *
38  * @author Phani Motamarri, Sambit Das, Krishnendu Ghosh
39  */
